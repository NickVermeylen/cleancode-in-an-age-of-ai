Clean Code Principles

1. Meaningful Names

Use descriptive and unambiguous names.

Avoid abbreviations and use intention-revealing identifiers.

Classes, methods, variables—everything should tell a story.

2. Small Functions

Functions should be small and do one thing.

Keep the number of arguments low.

Prefer expressive names over comments.

3. Code Structure

Organize code for readability: proper indentation, spacing, and ordering.

Maintain consistent formatting throughout the codebase.

4. Comments

Comments are not a substitute for clean code.

Use comments to explain why, not what—if the code is clear, it should explain itself.

Avoid redundant or outdated comments.

5. Objects and Data Structure

Use objects to hide implementation and expose behavior.

Data structures should expose data but not behavior.

Understand when to use one over the other.

6. Error Handling

Use exceptions rather than error codes.

Keep try/catch blocks to a minimum and handle them at a high level.

7. Boundaries

Use tests to explore third-party code or unfamiliar APIs.

Keep external dependencies well-encapsulated.

8. DRY Don't Repeat Yourself

Eliminate duplicate code.

Refactor repeated logic into well-named functions or classes.
